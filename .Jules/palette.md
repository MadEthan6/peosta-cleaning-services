## 2023-10-27 - Accessible Checkbox Toggles & Disabled Submit Buttons
**Learning:** Using a simple `div` element with an `onClick` handler as a custom checklist toggle is poor for keyboard accessibility and screen readers. When replacing them with `<button>` elements, styling properties like `textAlign: left`, `background: none`, and `border: none` make the button visually mimic the `div` layout while properly capturing focus. Also, adding `disabled` states to action buttons (like form submissions) should visually indicate their state (e.g., using `opacity: 0.5` and `cursor: not-allowed`) and include a `title` that informs the user *why* it's disabled.
**Action:** When implementing custom interactive elements like list item toggles, always use a `<button>` (not a `div`) and clear its default styling if needed to maintain layout. Include robust `aria-label` and `title` attributes indicating the resulting action of the element's current state. Apply both functional `disabled` attributes and corresponding visual styling (`opacity`, `cursor`) to actionable buttons to explicitly communicate form readiness.
