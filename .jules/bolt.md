## 2024-07-16 - Nominatim API Rate Limiting\n**Learning:** The OpenStreetMap Nominatim API has strict rate limits (1 request/second). Firing a request on every keystroke causes IP blocking and failed network requests in the frontend.\n**Action:** When working with third-party geolocation or search APIs, always wrap the network call in a debounce (e.g., 500ms using `setTimeout`). Make sure to clear the timeout at the very top of the handler (before any early returns based on length/emptiness) so rapid deletions correctly cancel pending calls.
