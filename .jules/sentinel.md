## 2024-05-15 - React Component Debouncing\n**Vulnerability:** API rate limiting risk (DoS vector) due to un-debounced third-party API calls in input fields. Failed fix attempt using local `let` variables for timeout IDs in React.\n**Learning:** In React components, defining timeout IDs with a local `let` variable fails because it re-initializes to `null` on every re-render (which happens on every keystroke). This effectively breaks the debounce logic and leaves the application vulnerable to rate limiting.\n**Prevention:** When debouncing operations inside functional React components, always store the timeout ID in a `useRef` hook (e.g., `const timeoutRef = React.useRef(null);`) to persist the reference across renders.
